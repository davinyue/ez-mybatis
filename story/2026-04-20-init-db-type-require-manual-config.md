# initDbType 未识别数据库类型时要求显式配置

## 背景

`EzMybatisContent.initDbType` 在未传入数据库类型且无法从数据源自动识别数据库类型时，会静默返回，导致问题延后到后续使用方言时才暴露。

## 目标

当未传入 `dbType` 且无法自动识别数据库类型时，初始化阶段直接抛出英文异常，提示使用方必须先设置数据库类型。

## 范围

本次仅调整 `EzMybatisContent.initDbType` 的失败路径，并补充核心模块单元测试覆盖。

## 非目标

不新增配置 API，不调整数据库方言匹配规则，不改变已显式传入 `dbType` 或可自动识别驱动时的正常初始化流程。

## 关键设计

在 `initDbType` 中，当 `EzMybatisConfig#getDbType` 为空时，依次尝试读取 MyBatis `Environment`、`DataSource` 和驱动类名。任一环节缺失、驱动名为空，或驱动无法匹配已有方言时，统一抛出英文异常：`Cannot recognize database type automatically. Please set the database type first`。

## 验收标准

未显式配置 `dbType` 且无法自动识别数据库类型时，`EzMybatisContent.init` 直接抛出上述英文异常。

显式配置 `dbType` 时，现有上下文初始化、方言隔离和销毁相关测试保持通过。
